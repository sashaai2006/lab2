## 2. Предметная область

В работе используется **вымышленное** расписание учебного потока для пяти групп физико-математического профиля.

### Допущения

| Параметр | Значение |
|---|---|
| Учебная неделя | 5 дней: Понедельник — Пятница |
| Временны́е слоты | 4 пары в день: 09:00, 10:45, 13:00, 14:45 |
| Длительность пары | 90 минут |
| Перерывы | учитываются только как атомы (длительности не вычисляются) |

### Инварианты расписания

1. У одной группы не более одной пары в один временной слот.
2. Один преподаватель не ведёт двух пар одновременно.
3. Одна аудитория не используется дважды в один слот.
4. Все атомы фактов принадлежат своим доменам.

Все четыре инварианта закодированы в `schedule.pl` отдельными предикатами (`конфликт_группы/5`, `конфликт_преподавателя/5`, `конфликт_аудитории/5`, `атом_вне_домена/2`) и проверяются автоматическими тестами в `tests.pl`.

## 4. Обязательные предикаты

### 4.1 `свободна_аудитория(+День, +Время, ?Аудитория)`

Истина, если в данный день в данное время в аудитории нет занятия. При незафиксированном третьем аргументе перебирает весь домен `аудитория/1`.

```prolog
свободна_аудитория(День, Время, Аудитория) :-
    аудитория(Аудитория),
    \+ занятие(_, _, _, День, Время, Аудитория).
```

Полный домен `аудитория/1` явно перечисляет все 10 аудиторий, поэтому предикат корректно находит свободные помещения, даже если в конкретный слот нет ни одного занятия в нужной аудитории.

---

### 4.2 `нагрузка_группы(?Группа, ?КоличествоПар)`

Подсчитывает общее число занятий у группы за учебную неделю через `findall/3` и `length/2`. Гарантия принадлежности группы домену защищает от обращений к несуществующим группам.

```prolog
нагрузка_группы(Группа, КоличествоПар) :-
    группа(Группа),
    findall(_, занятие(Группа, _, _, _, _, _), Список),
    length(Список, КоличествоПар).
```

При незаданном первом аргументе предикат перечисляет все группы; запрос `findall(N, нагрузка_группы(_, N), L)` возвращает `[10, 11, 12, 12, 10]`.

---

### 4.3 `окна_в_расписании(+Группа, +День)`

Истина, если у группы в указанный день есть хотя бы одно **окно** — пустой временной слот между двумя занятиями. Промежутки до первой пары и после последней пары окнами не считаются.

**Алгоритм:**

1. `findall` собирает номера занятых слотов через `порядок_времени/2`.
2. `sort/2` сортирует и удаляет дубликаты (дубликат был бы аномалией — ловится тестами).
3. `есть_пробел/1` рекурсивно проверяет каждую пару соседних номеров: если разность больше 1, между ними есть пустой слот → найдено окно.

```prolog
окна_в_расписании(Группа, День) :-
    группа(Группа),
    день(День),
    findall(Слот,
        ( занятие(Группа, _, _, День, Время, _),
          порядок_времени(Время, Слот) ),
        Слоты),
    sort(Слоты, ОтсортированныеСлоты),
    есть_пробел(ОтсортированныеСлоты).

есть_пробел([Прев, Тек | _]) :-
    Тек - Прев > 1, !.
есть_пробел([_, Тек | Хвост]) :-
    есть_пробел([Тек | Хвост]).
```

**Корректность по краям списка:**

| Слоты группы за день | Решение | Объяснение |
|---|---|---|
| `[]` | false | предикат `есть_пробел/1` не имеет факта для пустого списка |
| `[N]` | false | образец `[A, B \| _]` не сматчится |
| `[1, 2]` | false | разность 1, рекурсия съедает весь список |
| `[1, 3]` | **true** | разность 2 > 1 → окно |
| `[3, 4]` | false | разность 1, окон нет (хотя слоты 1-2 пустые — это «утренний пропуск», а не окно) |
| `[1, 2, 4]` | **true** | пара (2, 4) даёт разность 2 |

## 5. Тестовые запросы (`queries.pl`)

Каждый запрос — отдельный предикат `запрос_N/0` с осмысленным комментарием. Параметры запросов вынесены в верхнюю часть файла (`целевая_группа/1`, `целевой_предмет/1`, `целевой_преподаватель/1`, …) — чтобы менять «вход» запроса без редактирования его тела.

### Запрос 1. Когда у группы X предмет Y?

Все пары (день, время) для группы и предмета.

```
=== Запрос 1: Когда у группы Б24-501 предмет Математика? ===
  Понедельник, 09:00
  Пятница, 14:45
```

### Запрос 2. Какие аудитории свободны в среду в 10:45?

Использует `свободна_аудитория/3`.

```
=== Запрос 2: Свободные аудитории — Среда, 10:45 ===
  А101
  А102
  А103
  В301
  В302
  В303
  Г401
```

В этот слот заняты только Б201, Б202, Б203 — остальные 7 свободны.

### Запрос 3. У каких групп занятия ведёт преподаватель Иванов?

```
=== Запрос 3: Группы, у которых занятия ведёт Иванов ===
  Б24-501
  Б24-502
  Б24-503
  Б24-504
  Б24-505
```

### Запрос 4. Сколько пар в неделю у группы Б24-501?

Использует `нагрузка_группы/2`. Вспомогательный предикат `склонение_пар/2` подбирает корректное окончание русского числительного.

```
=== Запрос 4: Недельная нагрузка группы Б24-501 ===
  10 пар
```

### Запрос 5. Какие дни полностью свободны у преподавателя Козлов?

```
=== Запрос 5: Свободные дни преподавателя Козлов ===
  Четверг
```

### Запрос 6. Есть ли у группы Б24-501 окна?

Использует `окна_в_расписании/2`.

```
=== Запрос 6: Дни с окнами у группы Б24-501 ===
  Понедельник
  Вторник
  Среда
```

- Понедельник: слоты `{1, 3}` → пара (1, 3) разность 2 > 1.
- Вторник: слоты `{1, 2, 4}` → пара (2, 4) разность 2 > 1.
- Среда: слоты `{1, 3}` → разность 2 > 1.

### Запрос 7. В каких аудиториях идёт Математика?

```
=== Запрос 7: Аудитории, где проходит Математика ===
```

### Запрос 9. У каких групп в пятницу более двух пар?

```
  Б24-503 (3 пары)
```

## 6. Юнит-тесты (`tests.pl`)

В файле `tests.pl` через `plunit` реализовано **25 тестов**, проверяющих:

- инварианты базы (нет конфликтов группы / преподавателя / аудитории, нет атомов вне доменов);
- размер базы (5 групп, 10 предметов, 5 преподавателей, 10 аудиторий, 5 дней);
- работу `свободна_аудитория/3` в позитивном и негативном случаях;
- значения `нагрузка_группы/2` для конкретных групп и сумму нагрузок (= числу занятий);
- определение окон для всех краевых случаев (одно занятие, два соседних слота, два с пропуском, серия без окон);
- биективность `порядок_времени/2`;
- запуск всех 11 запросов без ошибок и исключений.

Запуск:

```bash
swipl -s tests.pl -g run_tests -t halt
```

Вывод (сокращённо):

```
% [1/25] schedule:нет_конфликтов_группы ............. passed
% [2/25] schedule:нет_конфликтов_преподавателя ...... passed
...
% [25/25] schedule:запросы_исполняются_без_ошибок ... passed
```

Все 25 тестов завершаются `passed`.

## 8. Выводы

В ходе работы была построена непротиворечивая база знаний расписания учебных занятий для пяти групп. Реализованы три обязательных предиката:

- `свободна_аудитория/3` — использует полный домен аудиторий и отрицание `\+`;
- `нагрузка_группы/2` — агрегирует данные через `findall/3`;
- `окна_в_расписании/2` — работает с упорядоченными номерами слотов, что позволяет корректно выявлять только внутренние пустые промежутки.

Дополнительно реализованы валидаторы инвариантов (`конфликт_*/5`, `атом_вне_домена/2`, `проверить_базу/0`) и юнит-тесты (`tests.pl`), которые автоматически проверяют корректность базы и предикатов.

Работа показала, что Prolog удобен для декларативного представления расписания: ограничения задаются как атомарные инварианты, а сложные запросы (поиск окон, агрегаты по преподавателям, выборка свободных аудиторий) реализуются компактными предикатами с использованием `findall/3`, `sort/2`, `forall/2` и отрицания `\+`. Отсутствие явных циклов и переменных-состояний делает код декларативным и легко проверяемым.
